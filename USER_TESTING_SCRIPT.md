# Light House Logistics - User Testing Script

A comprehensive guide for conducting user testing sessions to validate the food ordering platform.

---

## 👥 Test Participant Profile

### Target Users
- Age: 18-45
- Tech comfort level: Basic to advanced
- Food delivery app users
- Mobile and desktop users

### Recruitment Criteria
- Uses food delivery apps at least once a month
- Owns a smartphone and/or computer
- Lives in Lagos or similar urban area

---

## 📋 Pre-Test Setup

### Required Materials
- [ ] Laptop/desktop with application loaded
- [ ] Mobile device for responsive testing
- [ ] Notepad for observations
- [ ] Screen recording software (optional)
- [ ] Consent form
- [ ] Task cards
- [ ] Post-test questionnaire

### Test Environment
- Quiet room with minimal distractions
- Stable internet connection
- Application accessible at: `[YOUR_URL]`

---

## 🎯 Testing Objectives

1. Evaluate ease of account creation and login
2. Assess vendor browsing and search functionality
3. Test menu navigation and item selection
4. Validate cart management usability
5. Evaluate checkout process clarity
6. Assess order tracking comprehension
7. Measure overall satisfaction and trust

---

## 📝 Test Session Structure

### Introduction (5 minutes)

**Script:**
> "Thank you for participating in this test session. Today, we're testing a new food ordering platform called Light House Logistics. I want to emphasize that we're testing the platform, not you. There are no wrong answers or actions.
>
> Please think aloud as you complete the tasks - tell me what you're thinking, what you expect to happen, and any concerns you have. Your honest feedback will help us improve the experience.
>
> Do you have any questions before we begin?"

---

## 🎬 Test Scenarios

### Scenario 1: First-Time User Registration (10 minutes)

**Task:**
> "You've just heard about Light House Logistics from a friend and want to try ordering food. Create an account and explore what vendors are available."

**Success Criteria:**
- [ ] Successfully navigates to signup page
- [ ] Completes registration form
- [ ] Understands confirmation feedback
- [ ] Reaches vendor listing page

**Observations to Note:**
- Does user find the signup button easily?
- Are form fields clear and intuitive?
- Does password validation make sense?
- Is the transition to vendors page smooth?

**Probing Questions:**
- "What do you think of the signup process?"
- "Was anything unclear or confusing?"
- "How do you feel about the information requested?"

---

### Scenario 2: Browsing & Selecting a Vendor (8 minutes)

**Task:**
> "You're in the mood for Nigerian food. Find a vendor that serves Nigerian cuisine and explore their menu."

**Success Criteria:**
- [ ] Uses search or browses vendors
- [ ] Identifies cuisine type
- [ ] Clicks on vendor card
- [ ] Navigates to menu page

**Observations to Note:**
- How does user search for vendors?
- Do vendor cards provide enough information?
- Is the cuisine badge noticeable?
- Are ratings and delivery times clear?

**Probing Questions:**
- "How would you describe this vendor?"
- "What information is most important to you when choosing?"
- "Is anything missing that you'd like to see?"

---

### Scenario 3: Adding Items to Cart (10 minutes)

**Task:**
> "You've decided to order from this vendor. Add at least 2 different items to your cart, making sure you order enough to meet the minimum order requirement."

**Success Criteria:**
- [ ] Adds items to cart successfully
- [ ] Notices cart icon update
- [ ] Understands minimum order requirement
- [ ] Navigates to cart

**Observations to Note:**
- Is the "Add to Cart" button prominent?
- Does user notice cart count update?
- Does user check if minimum order is met?
- Are toast notifications helpful?

**Probing Questions:**
- "How do you know the item was added to your cart?"
- "What do you think about the minimum order requirement?"
- "Would you like to see cart contents without leaving this page?"

---

### Scenario 4: Managing Cart Items (5 minutes)

**Task:**
> "Review your cart. Change the quantity of one item and remove one item completely."

**Success Criteria:**
- [ ] Successfully updates item quantity
- [ ] Successfully removes an item
- [ ] Understands total price changes
- [ ] Feels confident about cart contents

**Observations to Note:**
- Are quantity controls intuitive?
- Is the remove button easy to find?
- Do price updates happen smoothly?
- Is the order summary clear?

**Probing Questions:**
- "Was it easy to modify your cart?"
- "Do you understand the pricing breakdown?"
- "Would you like any additional cart features?"

---

### Scenario 5: Checkout Process (10 minutes)

**Task:**
> "You're ready to order. Complete the checkout process up to the payment confirmation."

**Success Criteria:**
- [ ] Fills out delivery address
- [ ] Provides phone number
- [ ] Reviews order summary
- [ ] Understands payment method (OPay)
- [ ] Completes checkout

**Observations to Note:**
- Is form pre-filled with profile data?
- Are required fields clearly marked?
- Does user understand OPay payment?
- Is order summary visible and clear?

**Probing Questions:**
- "What do you think will happen when you click 'Pay'?"
- "Do you feel confident about your order details?"
- "Is the payment method clear to you?"

---

### Scenario 6: Payment Flow (5 minutes)

**Task:**
> "Observe what happens during payment processing."

**Success Criteria:**
- [ ] Understands payment is processing
- [ ] Waits for confirmation
- [ ] Reaches success or failure page
- [ ] Knows next steps

**Observations to Note:**
- Does loading state provide reassurance?
- Is wait time acceptable?
- Is success message clear?
- Are next steps obvious?

**Probing Questions:**
- "How did you feel during payment processing?"
- "What would you do next from this screen?"
- "Was the outcome clear?"

---

### Scenario 7: Order Tracking (8 minutes)

**Task:**
> "Find your order and check its status. Tell me what you understand about where your food is."

**Success Criteria:**
- [ ] Navigates to order history
- [ ] Finds specific order
- [ ] Accesses tracking page
- [ ] Understands current status

**Observations to Note:**
- Is order history easy to find?
- Are order statuses clear?
- Is tracking timeline intuitive?
- Does user know when to expect delivery?

**Probing Questions:**
- "What is the current status of your order?"
- "When do you expect your food to arrive?"
- "Is there anything you'd like to know that isn't shown?"

---

### Scenario 8: Profile Management (5 minutes)

**Task:**
> "Update your phone number in your profile."

**Success Criteria:**
- [ ] Finds profile page
- [ ] Locates phone number field
- [ ] Updates information
- [ ] Saves changes

**Observations to Note:**
- Is profile link easy to find?
- Is the form layout clear?
- Is save confirmation visible?

**Probing Questions:**
- "Was it easy to update your profile?"
- "What other information would you like to manage?"

---

## 📊 Post-Test Questionnaire

### User Satisfaction (1-5 scale)

1. **Overall Experience**
   - How satisfied are you with your overall experience?
   - 1 (Very Dissatisfied) → 5 (Very Satisfied)

2. **Visual Design**
   - How would you rate the visual design and aesthetics?
   - 1 (Very Poor) → 5 (Excellent)

3. **Ease of Use**
   - How easy was it to complete the tasks?
   - 1 (Very Difficult) → 5 (Very Easy)

4. **Trust & Confidence**
   - How much do you trust this platform with your order?
   - 1 (No Trust) → 5 (Complete Trust)

5. **Mobile Experience** (if tested)
   - How was your experience on mobile?
   - 1 (Very Poor) → 5 (Excellent)

### Open-Ended Questions

1. **What did you like most about Light House Logistics?**
   - (User response)

2. **What frustrated you or slowed you down?**
   - (User response)

3. **What would make you choose this over competitors like Jumia Food or Glovo?**
   - (User response)

4. **What features are missing that you'd expect to see?**
   - (User response)

5. **Would you recommend this to a friend?** (Yes/No)
   - Why or why not?

6. **On a scale of 1-10, how likely are you to use this platform?**
   - (User response)

### Comparative Questions

7. **Compared to other food delivery apps you've used:**
   - Better
   - About the same
   - Worse
   - Haven't used others

8. **The signup process was:**
   - Easier
   - About the same
   - Harder

9. **Finding what I wanted was:**
   - Easier
   - About the same
   - Harder

## 📈 Success Metrics

### Quantitative
- **Task completion rate:** Target 85%+
- **Time on task:** Compare to benchmarks
- **Error rate:** Target <15%
- **Clicks to complete:** Measure efficiency

### Qualitative
- **Satisfaction scores:** Target 4+ average
- **Net Promoter Score:** Target 7+ average
- **Positive sentiment:** Target 70%+
- **Feature requests:** Document all

---

## 🚨 Red Flags to Watch For

### Critical Issues
- User cannot complete signup
- Cart doesn't update
- Payment process fails
- User expresses security concerns
- User completely lost in navigation

### Medium Issues
- Confusion about vendor information
- Difficulty finding cart
- Unclear order status
- Form validation problems

### Minor Issues
- Cosmetic inconsistencies
- Minor text clarity
- Preferred feature variations

## 📋 Post-Session Tasks

### Immediately After
- [ ] Save all notes and recordings
- [ ] Rate severity of issues found
- [ ] Write up initial observations
- [ ] Flag critical issues for immediate attention

### Within 24 Hours
- [ ] Compile detailed findings report
- [ ] Create issue tickets for problems
- [ ] Share key insights with team
- [ ] Update testing metrics

### Within 1 Week
- [ ] Analyze patterns across multiple sessions
- [ ] Prioritize improvements
- [ ] Plan design iterations
- [ ] Schedule follow-up tests if needed

---

## 🔄 Iteration Testing

After implementing changes, run abbreviated tests focusing on:
- Previously problematic areas
- New features or changes
- Edge cases discovered
- Different user segments

## 💡 Tips for Moderators

### Do's
- ✅ Stay neutral and encouraging
- ✅ Ask "why" and "what were you expecting"
- ✅ Observe body language
- ✅ Take detailed notes
- ✅ Probe when users hesitate

### Don'ts
- ❌ Lead users to answers
- ❌ Defend the design
- ❌ Interrupt users
- ❌ Explain how things work
- ❌ Skip over user struggles
